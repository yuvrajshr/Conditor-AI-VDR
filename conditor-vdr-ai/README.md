# Conditor VDR AI — VDR Intelligence

An AI assistant that sits on top of a **Google Drive** virtual data room (VDR) and helps a private-equity deal team move faster through due diligence. Built for **Conditor Capital** (London growth-capital PE).

It is a real website with a real backend:

- **Frontend** — a single-page app (`index.html`, `app.js`) styled in Conditor's brand.
- **Backend** — a serverless function (`api/ai.js`) that calls a **free** AI model (Google Gemini) and keeps the API key on the server.
- **Google Drive** — connects to *your own* Drive (read-only) so it can work on real deal folders. Falls back to a built-in demo data room when not connected.

---

## What it does (six features)

1. **Overview** — AI summary of the folder hierarchy ("what's where").
2. **Navigate & Ask** — plain-English search that finds the right file and jumps to it.
3. **Summarise** — per-document AI summaries with a PE lens.
4. **Financial Extract** — maps statutory accounts into Conditor's internal **Adjusted EBITDA** template (computing EBITDA + add-backs that aren't shown in the source).
5. **Inconsistencies** — scans documents and flags risks / inconsistencies (with severity).
6. **Request Checklist** — reconciles your DD request list against the files in the room (missing vs. extra) and drafts a chaser email.

## Go live in ~15 minutes

You need two free things: a **Gemini API key** (for the AI) and a **Google OAuth Client ID** (for Drive). Then deploy to **Vercel** (free).

### Step 1 — Free AI key (Gemini)
1. Go to **https://aistudio.google.com/apikey** and sign in.
2. Click **Create API key** (no billing / credit card required for the free tier).
3. Copy the key — you'll paste it into Vercel in Step 3.

> Free tier is rate-limited (a handful of requests/minute) — plenty for a demo. To switch models, set `GEMINI_MODEL` (e.g. `gemini-2.5-flash-lite`).

### Step 2 — Deploy to Vercel (gives you a public URL + the backend)
1. Create a free account at **https://vercel.com** and install the GitHub app, **or** install the CLI: `npm i -g vercel`.
2. Push this folder to a GitHub repo (or run `vercel` from the CLI in this folder).
3. In Vercel → your project → **Settings → Environment Variables**, add:
   - `GEMINI_API_KEY` = the key from Step 1
   - *(optional)* `GEMINI_MODEL` = `gemini-2.5-flash`
4. **Deploy.** Vercel serves the static site and runs `api/ai.js` automatically at `/api/ai`. You now have a URL like `https://conditor-vdr-ai.vercel.app`.

At this point the **AI features are fully live** (they call your backend). Test "Generate room summary" on the demo data — it should say *"Generated by Gemini via Conditor backend"*.

### Step 3 — Connect your own Google Drive
1. Go to **https://console.cloud.google.com** → create/select a project.
2. **APIs & Services → Library → enable "Google Drive API".**
3. **APIs & Services → OAuth consent screen** → choose **External**, fill the basics, add the scope `.../auth/drive.readonly`, and add your own email under **Test users**.
4. **APIs & Services → Credentials → Create credentials → OAuth client ID → Web application.**
   - Under **Authorized JavaScript origins**, add your Vercel URL (e.g. `https://conditor-vdr-ai.vercel.app`) and, for local testing, `http://localhost:3000`.
   - Click **Create** and copy the **Client ID**.
5. Open `app.js`, find the top `CONFIG` block, and paste it:
   ```js
   GOOGLE_CLIENT_ID: "1234567890-abc.apps.googleusercontent.com",
   ```
   (Or set `window.CONDITOR_CLIENT_ID` in `index.html` before `app.js` loads.) Redeploy.
6. Open the site → **Connect Google Drive** (top right) → sign in → then paste a **folder link or ID** in the left panel to load that deal folder as the data room.

That's it — the same six tools now run on your real Drive files (Google Docs, Sheets, PDFs and text are read; scanned images aren't).

## Architecture

```
Browser (index.html + app.js)
   │  ├─ Google Identity Services + Drive API  ──►  your Google Drive (read-only)
   │  └─ POST /api/ai  ─────────────────────────►  Vercel serverless function (api/ai.js)
   │                                                      └─►  Gemini API (key stays server-side)
```

The AI key never reaches the browser — it lives in a Vercel environment variable and is only used inside `api/ai.js`. That's the point of having a backend.

---

## Swapping the AI provider (still free)

Edit `callGemini()` in `api/ai.js`. For example, **Groq** (free, very fast Llama models): point the fetch at `https://api.groq.com/openai/v1/chat/completions`, use `Authorization: Bearer ${process.env.GROQ_API_KEY}`, and the OpenAI-style `{messages:[...]}` body. The frontend doesn't change — it just calls `/api/ai`.

---

## Notes & limitations

- Read-only Drive access — the app never edits or deletes your files.
- Demo deal data (Project Meridian) is illustrative, not a real Conditor transaction.
- PDF reading is client-side text extraction (first ~15 pages); scanned PDFs without text won't extract.
- Free-tier rate limits apply; if you hit a 429, wait a moment and retry.
